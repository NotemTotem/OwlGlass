What are ports

Ports are the way we interact with services online such as HTTP/s, SSH, SMB. If we can scan the services on these ports we can sometimes interact with parts of the system that are home to vulnerabilities or sensitive data. A computer has 65535 ports when it is connected to the network, usually these ports are not open meaning that no service is running on them and we can not interact with the ports. A port scanner allows us to scan any ports we are interested in and use this information to discover different ways we can interact with a computer. This is part of increasing our attack surface.

What is an attack surface?

Think of an attack surface like the size of a target in a video game. The larger it is the easier more we can shoot at and the higher the chance we have of hitting the target. Our attack surface is the number of features, or services we can test to potentially attack a target. For example if our target has multiple web servers, multiple applications, or multiple network devices, these assets are all part of our attack surface.

Initially a larger attack surface might seem overwhelming but in practice it means that there is more code that could potentially have mistakes, or oversights. Most defensive security experts are focused on minimising the attack surface. They do this by ensuring that only the functionality that a user needs is what is publicly accessible. For example hosting a staff storage server on a publicly accessible IP address means that hackers could test for vulnerabilities and potentially find one. If they instead host it on a private network then hackers don't even have a chance to attack the staff server.

Increasing our attack surface

Before we can hack anything we need to try to generate a large attack surface and then systematically begin testing each part of that attack surface. Learning which parts of the attack surface are more likely to contain vulnerabilities is something that can only come with practice but you will develop naturally over time.
