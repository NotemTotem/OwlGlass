Subdomain and VHost Fuzzing

Learn how to discover new web functionalities and further expand our attack surface by fuzzing for subdomains and vhosts.

What is a HTTP request 

A HTTP request is the data that your browser sends every time it talks to a website. HTTP is a request/response protocol meaning every request you send to a server is met with a response. 




The start line contains information about the type of request we are sending, which part of the server we are sending it to, and what protocol we are using to send it. The number in the response here is called a response code and can be used to tell our browser whether the request was successful and what the server is doing with the request. 


Click this link to learn more about HTTP response codes


https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status


The headers contain information about the request or response itself. This is stuff like the length of our data, the type of data, where we are sending it from, and tokens that authenticate requests.


The body is where the actual data is sent. This might be login information for a website or the code for our browser to display a website.


What is fuzzing


Fuzzing is a testing methodology where we repeatedly send a request to a server but change a small part of the request each time. The part we change is called the payload, and where we place the payload can determine what type of fuzzing we are doing. The payload for each request is selected from a predetermined wordlist. A common set of wordlists for fuzzing can be found at https://github.com/danielmiessler/SecLists


In our case we change either the URL that we send the request to (subdomain fuzzer) or we change the host header (vhost fuzzing). 


What are subdomains and vhosts

Subdomains are ways that web apps can separate different functionalities. For example images.google.com is where you can find cat images on google, and maps.google.com is how you can find directions to the nearest place to adopt a kitten. 


There are two main ways that servers can do this:


Vhosts

Short for virtual host and this is one server that contains both subdomains and uses the “Host: “ header in the http request to tell which subdomain to send the request to. The payload for our fuzzing happens in this section for the vhost fuzzer.


Subdomains

Subdomains use different DNS records to point to a different IP address that has a separate server for that subdomain. To learn more about DNS 


The payload of our fuzzing is in the URL we send the request to for the subdomain fuzzer.
