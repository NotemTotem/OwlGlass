Glossary

Access-lists - Network rules that allow or deny traffic based on IPs, ports, or protocols.

API - A program which allows different applications to communicate and exchange data.

Payload - The variable part of a request (e.g., a filename or path) that the fuzzer replaces when probing a server.

Bug Bounty Program (BBP) - A program where organizations invite security researchers to find and report vulnerabilities, sometimes with payment.

Burp Suite - A web-application testing tool (interceptor/proxy and scanner used by testers).

Data breach / pwned - When account credentials or personal data are exposed publicly; “pwned” means included in such a breach.

Directory Fuzzing - Automated testing that requests many possible directory or file paths to discover hidden pages on a website.

DNS / Domain Name Service - The protocol/service that translates human-readable domain names to IP addresses.

DNS Lookup - Querying DNS servers to retrieve DNS records (A, MX, TXT, NS, etc.) for a domain.

Domain - A name registered on the Internet (e.g., google.com) that DNS maps to IPs and services.

Email / MX record - A DNS record that lists which mail servers accept email for a domain (e.g., smtp.google.com as a mail host).

Enterprise networks - Organizational networks that typically use firewalls and access-lists to protect internal resources.

Enumeration - Systematic discovery of hosts, directories, DNS records, accounts, or other information about a target or scope.

Firewall - Network device or software that filters traffic according to configured rules.

Fuzzing - Automated testing that feeds many inputs (e.g., file paths, subdomains) to a service to discover unexpected behavior or hidden resources.

Have I Been Pwned - A public service to check whether an email or account appears in known data breaches.

HackerOne / Bugcrowd - Platforms that host bug bounty programs where researchers report vulnerabilities to organizations.

IP address - Numerical address that identifies a host on a network (e.g., 15.197.167.90).

Kali Linux / Parrot OS - Linux distributions commonly used for penetration testing and security research.

Nmap - Network scanning tool used to discover hosts, ports, and services.

OSINT (Open Source Intelligence) - Collecting information from publicly available sources (e.g., DNS, social media, public records).

Organization profile - Building a profile of a person or organization from public data, often to find accounts or contacts.

Payload - The part of a request that changes during fuzzing.

Phishing / Social engineering - Techniques that manipulate people into revealing information or performing actions.

Portswigger Web Security Academy - Learning platform for web application security training.

Public information - Data that is publicly accessible (e.g., DNS records, social posts, sitemaps).

Robots.txt - A web file that tells web crawlers which pages to avoid indexing.

Scanning - The act of probing systems (e.g., with Nmap or directory fuzzers) to identify services and responses.

Scope - The set of systems, networks, applications, domains, or other assets that are authorized for testing or attack during a penetration test. 

Sitemap.xml - An XML file listing pages on a site to help search engines index content.

smtp.google.com - Example mail host used in the activity to demonstrate MX records.

Subdomain - A subsection of a domain which contains its own unique functionality.

Target - The domain, URL, or email chosen for testing (e.g., google.com or your email in HaveIBeenPwned).

TryHackMe / Hack The Box Academy - Practical labs for developing offensive security skills.

Vulnerability Disclosure Program (VDP) - A process for reporting vulnerabilities for potential rewards.

Web directories - Collections of pages and paths on a website (analogous to a filesystem).

Well-known / .well-known/security.txt - A standardized path (.well-known/security.txt) where a domain can publish security contact information.


What is cybersecurity?


Cybersecurity is the practice of protecting networks, applications, websites and data from malicious actors. There are many branches of cybersecurity with the main two being offensive and defensive often referred to as red and blue teaming. 


Offensive security or red teaming is the act of attempting to find and exploit misconfigurations and logic flaws in the assets of an application or network. The target of this act is to find sensitive and valuable data, not otherwise revealed to the public. Numerous techniques such as SQL injections, XSS, command injections, request forgeries, certificate forgeries and many more can be used to gain unauthorized access to networks and applications, resulting in data breaches and hostage situations. Therefore, enterprises will often have their own team of offensive security experts in order to try and find these vulnerabilities bad actors are able to, and thus protect themselves from attacks. 


Defensive cybersecurity or blue teaming is the act of strengthening your data security and making it harder for attackers to find and exploit vulnerabilities. This includes analysing network traffic to monitor for potential attacks, administrating domains and servers in organisations, and much more. Defensive cybersecurity experts are also often required to respond to cyberattacks and patch any vulnerable services immediately after an attack to limit the impact on the company. It is important to understand the basics of the defensive team in order to become a successful attacker, and without the blue team the world net would be full of malicious threat actors and stolen data.


In our learning journey, we will be focusing on the offensive side of cybersecurity with lessons centered around reconnaissance, profiling and enumeration.
